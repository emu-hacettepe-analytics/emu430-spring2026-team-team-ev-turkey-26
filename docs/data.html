🏗️ Data Source

To ensure transparency and reliability, the foundational dataset for our project was compiled from a unified official source:

  • EPDK (Energy Market Regulatory Authority): Provincial-level data containing the number of registered electric vehicles and the geographical distribution of charging stations (sockets).

🛠️ Preprocessing & “Fuzzy Matching”

The most significant challenge we faced while merging raw datasets was the formatting of city names (e.g., encoding errors, typos, HTML tags).

To solve this without introducing human error, we built a “City Repair” Engine:

  • The Problem: Corrupted data entries like “Afyonkara” or “Istanbul ”.
  • The Solution: Fuzzy Matching Algorithm.
  • The Tool: The stringdist package in R.
TipTechnical Insight

Every provincial entry was automatically cross-referenced and corrected against an official benchmark list of 81 provinces using a maximum distance metric (amatch). This ensured 100% data integrity before modeling.

The following is our reproducible code pipeline for cleaning the datasets:

official_cities <- c(

  "Eskişehir", "Gaziantep", "Giresun", "Gümüşhane", "Hakkari", "Hatay", "Isparta", "Mersin",
  "İstanbul", "İzmir", "Kars", "Kastamonu", "Kayseri", "Kırklareli", "Kırşehir", "Kocaeli",
  "Konya", "Kütahya", "Malatya", "Manisa", "Kahramanmaraş", "Mardin", "Muğla", "Muş",
  "Nevşehir", "Niğde", "Ordu", "Rize", "Sakarya", "Samsun", "Siirt", "Sinop", "Sivas",
  "Tekirdağ", "Tokat", "Trabzon", "Tunceli", "Şanlıurfa", "Uşak", "Van", "Yozgat", "Zonguldak",
  "Aksaray", "Bayburt", "Karaman", "Kırıkkale", "Batman", "Şırnak", "Bartın", "Ardahan",
  "Iğdır", "Yalova", "Karabük", "Kilis", "Osmaniye", "Düzce"
)

# 4. CITY REPAIR ENGINE (Fuzzy Matching)
clean_and_match_city <- function(messy_city) {
  # Clean up garbage characters first
  cleaned <- messy_city %>%
    str_replace_all("&[[:alpha:]]+;", "") %>% # Remove HTML entities
    str_replace_all("[^[:alnum:] ]", " ") %>% # Remove non-alphanumeric characters
    str_replace_all("Ã", "Ç") %>%             # Pre-cleaning for common encoding errors
    str_to_lower(locale = "tr") %>%
    str_squish()
  
  # If the city name is highly corrupted, find the closest official name
  match_idx <- amatch(cleaned, str_to_lower(official_cities, locale = "tr"), maxDist = 5)
  
  if (!is.na(match_idx)) {
    return(official_cities[match_idx])
  } else {
    return(str_to_title(cleaned, locale = "tr"))
  }
}

# 5. File Processing
process_file <- function(f) {
  tarih_val <- str_extract(basename(f), "\\d{4}_\\d{2}")
  
  # Encoding-independent read attempt
  d <- read.csv(f, stringsAsFactors = FALSE, check.names = FALSE, fileEncoding = "UTF-8")
  if (ncol(d) < 5) d <- read.csv(f, stringsAsFactors = FALSE, check.names = FALSE, fileEncoding = "CP1254")
  
  # Force assign column names
  if (ncol(d) == 5) {
    names(d) <- c("sehir", "istasyon", "otomobil", "ev", "oran")
  } else {
    names(d) <- c("sehir", "istasyon", "degisim", "otomobil", "ev", "oran")
  }
  
  d %>%
    as_tibble() %>%
    mutate(
      tarih = tarih_val,
      # Compare and fix each city against the official list
      sehir = map_chr(sehir, clean_and_match_city),
      otomobil_sayisi = as.numeric(str_remove_all(otomobil, "[^0-9]")),
      ev_sayisi = as.numeric(str_remove_all(ev, "[^0-9]")),
      istasyon_sayisi = as.numeric(str_remove_all(istasyon, "[^0-9]")),
      istasyon_doluluk_orani = as.numeric(str_replace_all(oran, ",", "."))
    ) %>%
    select(tarih, sehir, otomobil_sayisi, ev_sayisi, istasyon_sayisi, istasyon_doluluk_orani)
}

# 6. Merge and Save
final_df <- file_list %>%
  map_dfr(process_file) %>%
  arrange(sehir, tarih) %>%
  distinct(tarih, sehir, .keep_all = TRUE)

-   **The Problem:** Corrupted data entries like "Afyonkara" or "Istanbul ".

-   **The Tool:** The `stringdist` package in R.

### Technical Insight
